Single Coupon Screenshot Upload Flow

📱 User Action
HomeScreen → SimplifiedCaptureBottomSheet
User clicks "Upload" button and selects single image from gallery
↓
🎬 ViewModel Entry
CouponFormViewModel.processImageUri()
• Receives image URI
• Calls couponInputManager.processCouponFromImageUriWithPersistence()
↓
🔧 Input Manager
CouponInputManager.processCouponFromImageUri()
• Persists URI to app storage
• Extracts capture timestamp from image metadata
• Loads bitmap from URI
• Calls processCouponFromBitmap()
↓
🔍 Classification Step
ScreenshotClassifier.classify()
• Runs quick OCR (QuickOCREngine)
• Analyzes text for multi-coupon patterns
• Detects if image contains multiple coupons
↓
🎯 If Multi-Coupon Detected
Multi-Coupon Extraction
MultiCouponExtractionService
1. Full OCR (MultiEngineOCR)
2. Detect coupon regions (HybridCouponDetector)
3. Extract each region separately
4. Returns list of coupons
↓
⚠️ Combine Results
handleMultiCouponResult()
Returns FIRST coupon only to user
(Others are lost - not saved to database)
📄 If Single Coupon
1️⃣ Barcode Scan
BarcodeScanner
Scans for QR codes/barcodes
If URL found → process URL
If text found → use as coupon code
↓
2️⃣ OCR Extraction
LocalLlmOcrService
• Runs ML Kit Text Recognition
• Extracts all text from image
• Timeout: 30 seconds
↓
3️⃣ LLM Field Extraction
StructuredFieldExtractor (MiniCPM)
• Sends OCR text to on-device LLM (llama.cpp)
• Model: MiniCPM-2.5B-Q4
• Extracts: store, code, amount, expiry
• First run: ~60s (model warmup)
• Subsequent: ~10s
• Timeout: 90 seconds
↓
⚠️ On Timeout/Failure
Fallback Logic
Falls back to pattern matching:
• Regex for dates
• Regex for discount amounts
• Heuristics for store names
❌ Can produce incorrect results
↓
✅ Final Output
Coupon object returned to ViewModel
Coupon fields populated:
• storeName, redeemCode, discountAmount, expiryDate
• imageUri (persisted path)
• captureTimestamp, createdAt
• Displayed in CouponFormScreen for user review
🔍 Critical Issues in Single Coupon Flow:
  • Multi-coupon data loss: If multiple coupons detected, only first one is returned. Others are discarded.
  • Timeout fallback: 90s timeout on LLM can still trigger fallback to pattern matching if LLM is slow.
  • No retry mechanism: If LLM fails, immediately falls back to pattern matching instead of retrying.
  • Pattern matching contradicts goal: Goal was "use LLM for accurate extraction", but timeout defeats this.

Multiple Coupons in One Image Flow

📱 User Action
HomeScreen → Upload single screenshot with multiple coupons
User uploads image containing multiple coupon cards (e.g., from coupon aggregator app)
↓
🎬 Entry Point
CouponInputManager.processCouponFromBitmap()
Same as single coupon flow
↓
🔍 Step 1: Quick OCR
QuickOCREngine.recognize()
Fast text extraction for classification:
• Uses ML Kit Text Recognition
• Extracts all visible text
↓
🎯 Step 2: Classification
ScreenshotClassifier.classify()
Analyzes text patterns to detect:
• MULTI_COUPON_APP (coupon aggregator)
• E_COMMERCE_CART (checkout page)
• EMAIL_SCREENSHOT (promotional email)
• CAMERA_CAPTURE (single coupon)

Returns confidence score + coupon count estimate
↓
🚀 Step 3: Multi-Coupon Pipeline
MultiCouponExtractionService.extractMultipleCoupons()
Main orchestrator for multi-coupon extraction
↓
📝 Step 3a: Full OCR
MultiEngineOCR.processImage()
Comprehensive text extraction:
• ML Kit Text Recognition
• Returns structured OCR result with text blocks
• Preserves spatial layout information
↓
🎯 Step 3b: Region Detection
HybridCouponDetector.detectCoupons()
Detects individual coupon regions:
• Uses OCR text block clustering
• Spatial analysis of coupon patterns
• Returns bounding boxes for each coupon
• Max: 10 coupons per image
↓
✂️ Step 3c: Crop & Extract Each Region
Loop through detected regions
For each detected coupon region:
1. Crop bitmap to bounding box
2. Run OCR on cropped region
3. Extract fields using StructuredFieldExtractor
4. Create Coupon object
5. Add to results list
↓
🧹 Step 3d: Filter & Deduplicate
QualityFilter & CouponDedupUtils
• Remove low-quality extractions
• Filter out "Unknown Store" results
• Deduplicate based on code similarity
• Keep only valid coupons
↓
📦 Step 4: Return Results
MultiCouponResult
Returns:
• List of extracted coupons (0-10)
• Screenshot type classification
• Statistics: detected, extracted, filtered counts
↓
⚠️ Step 5: Combine/Select
handleMultiCouponResult()
CRITICAL ISSUE:
Only returns FIRST coupon to user!
Remaining coupons are discarded.

User sees form with 1 coupon, not all extracted coupons.
🚨 Critical Issues in Multi-Coupon Flow:
  • Data Loss: System successfully extracts N coupons but only shows user 1 coupon. This defeats the entire purpose of multi-coupon detection!
  • No batch save UI: UI (CouponFormScreen) is designed for single coupon entry, not batch review.
  • Wasted processing: All the work to detect, crop, and extract multiple regions is thrown away.
  • User confusion: User uploads image with 5 coupons, sees only 1 extracted. No indication that others were detected.
  • Missing integration: MultiCouponExtractionService works correctly but isn't properly integrated into UI layer.

Multiple Screenshots Batch Upload Flow

📱 User Action
HomeScreen → "Screenshot Upload" (Batch Mode)
User selects multiple images (2-50) from gallery
Intent: Bulk import screenshots of coupons
↓
🎬 Batch Entry Point
BatchScannerViewModel.processImages()
• Receives list of image URIs
• Checks if TwoStageDetector is available
• Starts batch processing loop
↓
🔄 For Each Image
Loop: images.forEach { processImage(it) }
Processes images sequentially (not in parallel)
Updates progress UI after each image
↓
🖼️ Load Image
BitmapFactory.decodeStream()
• Loads bitmap from URI
• Tracks bitmap in BitmapManager
• Validates mime type
↓
🔍 Detect & Extract Multiple Coupons
detectAndExtractMultipleCoupons()
Same multi-coupon pipeline as Tab 2:
1. Run full OCR
2. Detect coupon regions
3. Extract each region
4. Returns list of coupons per image
↓
📝 OCR Each Image
MultiEngineOCR.processImage()
Runs ML Kit OCR on full image
↓
🎯 Detect Regions
HybridCouponDetector.detectCoupons()
Finds coupon regions within each image
(Each image may contain 1 or more coupons)
↓
✂️ Extract Each Region
extractCouponFromRegion()
For each detected region:
• Crop to bounding box
• Run OCR on region
• Extract fields using StructuredFieldExtractor
• Create Coupon object
↓
📊 Accumulate Results
processedCoupons.addAll(imageCoupons)
• Adds all extracted coupons to results list
• Tracks success/failure per image
• Updates progress counter
↓
🧹 Cleanup Bitmap
bitmapManager.releaseBitmap()
Releases bitmap memory after processing
↓
✅ Batch Results
Navigate to BatchResultsScreen
• Shows list of ALL extracted coupons
• User can review, edit, and select which to save
• Statistics: X images → Y coupons extracted
• Failed images shown with error messages
↓
💾 Bulk Save
User confirms → saveCoupons()
• User reviews list and selects coupons to keep
• Selected coupons saved to database
• Deduplication applied
• Success confirmation shown
✅ What Works Well in Batch Flow:
  • Complete extraction: All coupons from all images are extracted and shown to user.
  • Batch review UI: BatchResultsScreen properly displays all extracted coupons.
  • User control: User can review, edit, or discard individual coupons before saving.
  • Multi-coupon support: Each image can contain multiple coupons, all are extracted.
  • Progress tracking: Real-time progress updates during batch processing.
⚠️ Issues in Batch Flow:
  • Sequential processing: Images processed one-by-one, not in parallel. Can be slow for large batches.
  • No pause/resume: If user exits during processing, progress is lost.
  • Memory pressure: Processing many large images can cause memory issues on low-end devices.
  • No retry on failure: If one image fails, it's skipped with no retry option.
